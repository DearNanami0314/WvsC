# 自研卡牌项目：文件职责与关系（详细版）

> 面向目标：
> 1) 每个文件做什么（功能职责）
> 2) 每个文件和谁有关系（调用/继承/信号）
>
> 说明：这是**架构设计稿**，关系为“计划关系”，用于你后续实现时对齐。

---

## A. 一页版总览（你随时先看这个）

```text
输入（玩家操作/敌人AI）
  -> BattleController（战斗总编排）
     -> TurnManager（阶段推进）
     -> PlayerController / EnemyController（双方执行）
        -> Deck/Hand 系统（抽牌/弃牌/计数）
        -> PatternEngine（识别牌型）
        -> SkillResolver（结算技能效果）
           -> DamageRule / DefeatRule（规则落地）
              -> Events（广播UI刷新/回合切换/胜负）
                 -> UI层（手牌/提示/意图/日志）
```

---

## B. 执行版（按目录逐文件）

字段说明：
- **功能**：这个文件负责什么。
- **调用关系**：它主动调用谁；谁会调用它。
- **继承关系**：继承自谁（Godot基类/自定义基类）。
- **信号关系**：发哪些信号、监听哪些信号。

---

## 1) global/

### `global/events.gd`
- 功能：全局事件总线。
- 调用关系：被所有系统引用（Battle/UI/Deck/AI）。
- 继承关系：`Node`。
- 信号关系：
  - 发送源：`battle_controller.gd`、`turn_manager.gd`、`player_controller.gd`、`enemy_controller.gd`。
  - 监听方：`battle_hud`、`hand_ui`、`intent_ui`、`deck_count_widget`、`log_panel`。

### `global/game_config.gd`
- 功能：全局参数（初始抽牌数、手牌上限、MVP规则开关）。
- 调用关系：被 `turn_manager.gd`、`draw_system.gd`、`defeat_rule.gd` 读取。
- 继承关系：`Resource` 或 `Node`（二选一，推荐Resource）。
- 信号关系：无（纯配置）。

### `global/audio_manager.gd` [V1]
- 功能：统一播放BGM/SFX。
- 调用关系：被 `battle_controller.gd`、`skill_resolver.gd` 调用。
- 继承关系：`Node`。
- 信号关系：可监听 `Events.card_played`/`Events.battle_ended`。

### `global/logger.gd` [V1]
- 功能：统一战斗日志。
- 调用关系：被结算层与战斗层调用。
- 继承关系：`Node`。
- 信号关系：向 `log_panel` 推送日志更新事件。

---

## 2) core/turn/

### `core/turn/phase.gd`
- 功能：阶段枚举与阶段合法迁移定义。
- 调用关系：被 `turn_manager.gd` 使用。
- 继承关系：无（常量脚本）或 `RefCounted`。
- 信号关系：无。

### `core/turn/turn_manager.gd`
- 功能：回合与阶段状态机（玩家/敌方切换）。
- 调用关系：
  - 调用：`phase.gd`、`player_controller.gd`、`enemy_controller.gd`。
  - 被调用：`battle_controller.gd`。
- 继承关系：`Node`。
- 信号关系：
  - 发送：`turn_started`、`phase_changed`、`turn_ended`（可通过 `Events` 转发）。
  - 监听：`player_action_done`、`enemy_action_done`。

---

## 3) core/pattern/

### `core/pattern/pattern_types.gd`
- 功能：牌型枚举与比较优先级。
- 调用关系：被 `pattern_engine.gd`、`skill_resolver.gd` 使用。
- 继承关系：常量脚本。
- 信号关系：无。

### `core/pattern/pattern_engine.gd` [MVP]
- 功能：识别单张/对子/三条。
- 调用关系：
  - 调用：`pattern_types.gd`。
  - 被调用：`player_controller.gd`、`pattern_hint_ui`。
- 继承关系：`Node` 或 `RefCounted`（推荐RefCounted）。
- 信号关系：可选发送 `pattern_detected`。

### `core/pattern/pattern_engine_ext.gd` [V1]
- 功能：扩展顺子/同花/葫芦/四条。
- 调用关系：被 `pattern_engine.gd` 组合调用。
- 继承关系：`RefCounted`。
- 信号关系：无。

---

## 4) core/rules/

### `core/rules/card_play_rule.gd`
- 功能：验证当前阶段/当前手牌是否可出。
- 调用关系：被 `player_controller.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：无。

### `core/rules/damage_rule.gd` [MVP A版]
- 功能：受伤=弃置就绪牌堆等量卡牌。
- 调用关系：被 `skill_resolver.gd`、`enemy_controller.gd` 调用；调用 `discard_system.gd`。
- 继承关系：`RefCounted`。
- 信号关系：发送 `Events.player_damaged` / `Events.enemy_damaged`。

### `core/rules/defeat_rule.gd` [MVP]
- 功能：判负条件判断（抽牌失败/资源归零）。
- 调用关系：被 `turn_manager.gd`、`draw_system.gd`、`battle_controller.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：发送 `Events.battle_ended`。

### `core/rules/advanced_damage_rule.gd` [V2]
- 功能：销毁/精神伤害/DEBUFF标记。
- 调用关系：后续替换/扩展 `damage_rule.gd`。
- 继承关系：`RefCounted`。
- 信号关系：发送 `Events.status_applied`。

---

## 5) core/resolver/

### `core/resolver/skill_resolver.gd` [MVP]
- 功能：把“牌型+技能配置”转成具体效果。
- 调用关系：
  - 调用：`damage_rule.gd`、`target_selector.gd`（V1）、`effect_pipeline.gd`（V1）。
  - 被调用：`player_controller.gd`、`enemy_controller.gd`。
- 继承关系：`Node` 或 `RefCounted`。
- 信号关系：发送 `Events.skill_resolved`。

### `core/resolver/effect_pipeline.gd` [V1]
- 功能：效果执行顺序管线（伤害→护甲→状态→日志）。
- 调用关系：被 `skill_resolver.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：发送阶段性结算事件。

### `core/resolver/target_selector.gd` [V1]
- 功能：选择目标（自身/单体/群体）。
- 调用关系：被 `skill_resolver.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：无。

---

## 6) data/resources/

### `data/resources/card_data.gd`
- 功能：单卡静态定义（花色、点数、标签、费用、效果ID）。
- 调用关系：被 `hand_manager.gd`、`pattern_engine.gd`、`skill_resolver.gd` 读取。
- 继承关系：`Resource`。
- 信号关系：无。

### `data/resources/character_data.gd`
- 功能：角色基础属性、初始技能组、初始牌组。
- 调用关系：被 `battle_controller.gd`、`player_controller.gd` 初始化读取。
- 继承关系：`Resource`。
- 信号关系：无。

### `data/resources/enemy_data.gd`
- 功能：敌人属性、AI参数、动作池。
- 调用关系：被 `enemy_controller.gd`、`enemy_ai_base.gd` 读取。
- 继承关系：`Resource`。
- 信号关系：无。

### `data/resources/skill_data.gd`
- 功能：技能定义（触发牌型、目标、效果参数）。
- 调用关系：被 `skill_resolver.gd` 读取。
- 继承关系：`Resource`。
- 信号关系：无。

### `data/resources/status_data.gd` [V1]
- 功能：状态定义（持续回合、叠层规则）。
- 调用关系：被 `effect_pipeline.gd`、`player_controller.gd` 读取。
- 继承关系：`Resource`。
- 信号关系：无。

---

## 7) battle/

### `battle/battle_controller.gd` [MVP]
- 功能：单场战斗总导演（初始化、流程推进、胜负收口）。
- 调用关系：
  - 调用：`turn_manager.gd`、`player_controller.gd`、`enemy_controller.gd`、`defeat_rule.gd`。
  - 被调用：`game_root` 场景。
- 继承关系：`Node2D` 或 `Node`。
- 信号关系：
  - 监听：`Events.battle_ended`。
  - 发送：`Events.battle_started`、`Events.battle_ended`。

### `battle/player_controller.gd` [MVP]
- 功能：玩家在战斗中的行为入口。
- 调用关系：
  - 调用：`draw_system.gd`、`hand_manager.gd`、`pattern_engine.gd`、`card_play_rule.gd`、`skill_resolver.gd`。
  - 被调用：`turn_manager.gd`、UI输入。
- 继承关系：`Node`。
- 信号关系：发送 `Events.player_action_done`、`Events.cards_drawn`。

### `battle/enemy_controller.gd` [MVP]
- 功能：敌方回合执行与动作调度。
- 调用关系：
  - 调用：`enemy_ai_base.gd`、`skill_resolver.gd`。
  - 被调用：`turn_manager.gd`。
- 继承关系：`Node`。
- 信号关系：发送 `Events.enemy_action_done`。

### `battle/ai/enemy_ai_base.gd`
- 功能：敌人AI接口/基类（choose_action）。
- 调用关系：被 `enemy_controller.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：无。

### `battle/ai/simple_weight_ai.gd` [MVP]
- 功能：按权重选动作（简化AI）。
- 调用关系：被 `enemy_controller.gd` 实例化与调用。
- 继承关系：`enemy_ai_base.gd`。
- 信号关系：无。

### `battle/ai/interrupt_ai.gd` [V2]
- 功能：Boss时间条打断逻辑。
- 调用关系：被 `enemy_controller.gd` 在Boss战调用。
- 继承关系：`enemy_ai_base.gd`。
- 信号关系：发送 `Events.interrupt_triggered`。

### `battle/state/battle_state.gd`
- 功能：战斗快照（双方资源、阶段、随机种子）。
- 调用关系：被 `battle_controller.gd` 持有。
- 继承关系：`Resource` 或 `RefCounted`。
- 信号关系：无。

### `battle/state/action_queue.gd` [V1]
- 功能：动作队列（顺序执行/回放）。
- 调用关系：被 `battle_controller.gd`、`effect_pipeline.gd` 调用。
- 继承关系：`Node`。
- 信号关系：发送 `action_enqueued`/`action_executed`。

## 8) deck/

### `deck/deck_manager.gd` [MVP]
- 功能：统一管理就绪牌堆与弃牌堆。
- 调用关系：调用 `draw_system.gd`、`discard_system.gd`；被 `player_controller.gd` 调用。
- 继承关系：`Node`。
- 信号关系：发送 `Events.deck_count_changed`。

### `deck/draw_system.gd` [MVP]
- 功能：抽牌实现（不足时触发判负检查）。
- 调用关系：被 `player_controller.gd`、`turn_manager.gd` 调用；调用 `defeat_rule.gd`。
- 继承关系：`RefCounted`。
- 信号关系：发送 `Events.card_drawn`、`Events.draw_failed`。

### `deck/discard_system.gd` [MVP]
- 功能：弃牌实现（主动弃牌/受伤弃牌）。
- 调用关系：被 `player_controller.gd`、`damage_rule.gd` 调用。
- 继承关系：`RefCounted`。
- 信号关系：发送 `Events.card_discarded`。

### `deck/hand_manager.gd` [MVP]
- 功能：手牌增删、手牌上限、手牌排序。
- 调用关系：被 `player_controller.gd`、`hand_ui` 调用。
- 继承关系：`Node`。
- 信号关系：发送 `Events.hand_changed`。

### `deck/reserve_deck_manager.gd` [V1]
- 功能：备牌堆（战斗外替换）。
- 调用关系：被 `node_select` 或 `event_panel` 调用。
- 继承关系：`Node`。
- 信号关系：发送 `Events.reserve_updated`。

## 9) scenes/ 与 ui/

### `scenes/main/game_root.tscn`
- 功能：根场景，装配全局节点与流程入口。
- 调用关系：实例化 `battle.tscn`。
- 继承关系：场景。
- 信号关系：连接全局开始事件。

### `scenes/battle/battle.tscn`
- 功能：战斗主场景。
- 调用关系：挂载 `battle_controller.gd`。
- 继承关系：场景。
- 信号关系：订阅并分发战斗UI更新。

### `scenes/battle/battle_hud.tscn`
- 功能：HUD总容器（手牌、提示、意图、计数）。
- 调用关系：读取 `Events` 与控制器状态。
- 继承关系：场景。
- 信号关系：监听 `hand_changed`、`phase_changed`、`deck_count_changed`。

### `scenes/card/card_ui.tscn`
- 功能：单张卡可交互UI。
- 调用关系：被 `hand_ui.tscn` 实例化。
- 继承关系：场景（脚本建议继承 `Control`）。
- 信号关系：发送 `card_selected`、`card_play_requested`。

### `scenes/card/hand_ui.tscn`
- 功能：手牌区。
- 调用关系：调用 `hand_manager.gd`；被 `battle_hud` 管理。
- 继承关系：场景。
- 信号关系：监听 `Events.hand_changed`。

### `scenes/card/pattern_hint_ui.tscn`
- 功能：实时显示当前可组成牌型。
- 调用关系：调用 `pattern_engine.gd` 或接收其结果。
- 继承关系：场景。
- 信号关系：监听 `card_selected`。

### `scenes/card/card_state_machine.gd`
- 功能：卡牌交互状态（待选/拖拽/释放）。
- 调用关系：被 `card_ui.tscn` 调用。
- 继承关系：`Node`。
- 信号关系：发送状态切换事件给 `Events`。

### `scenes/player/player_actor.tscn`
- 功能：玩家表现体（动画/受击反馈）。
- 调用关系：被 `battle.tscn` 挂载。
- 继承关系：场景。
- 信号关系：监听 `player_damaged`。

### `scenes/enemy/enemy_actor.tscn`
- 功能：敌人表现体（动画/受击反馈）。
- 调用关系：被 `enemy_controller.gd` 驱动。
- 继承关系：场景。
- 信号关系：监听 `enemy_damaged`。

### `scenes/enemy/intent_ui.tscn`
- 功能：显示敌人下一动作意图。
- 调用关系：数据来自 `enemy_controller.gd`。
- 继承关系：场景。
- 信号关系：监听 `enemy_intent_updated`。

### `ui/widgets/deck_count_widget.tscn`
- 功能：显示就绪/弃牌数量。
- 调用关系：读取 `deck_manager.gd` 或监听事件。
- 继承关系：场景。
- 信号关系：监听 `deck_count_changed`。

### `ui/widgets/log_panel.tscn` [V1]
- 功能：显示战斗日志。
- 调用关系：读取 `logger.gd`。
- 继承关系：场景。
- 信号关系：监听 `log_updated`。

### `ui/widgets/status_bar.tscn` [V1]

- 调用关系：读取状态系统。
- 继承关系：场景。
- 信号关系：监听 `status_applied`、`status_expired`。

## C. 核心关系图（调用/继承/信号）

### 1) 调用主链（MVP）

```text
battle_controller
  -> turn_manager
    -> player_controller
      -> draw_system -> defeat_rule
      -> hand_manager
      -> pattern_engine
      -> card_play_rule
      -> skill_resolver -> damage_rule -> discard_system
    -> enemy_controller
      -> simple_weight_ai
      -> skill_resolver
```

### 2) 继承主链（MVP）

```text
simple_weight_ai : enemy_ai_base
interrupt_ai     : enemy_ai_base (V2)
card_data        : Resource
character_data   : Resource
enemy_data       : Resource
skill_data       : Resource
```

### 3) 信号主链（MVP）

```text
player_controller -> Events.cards_drawn / player_action_done
enemy_controller  -> Events.enemy_action_done / enemy_intent_updated
deck_manager      -> Events.deck_count_changed
draw_system       -> Events.card_drawn / draw_failed
discard_system    -> Events.card_discarded
turn_manager      -> Events.phase_changed / turn_started / turn_ended
defeat_rule       -> Events.battle_ended
UI层              <- 监听上述事件并刷新
```

## D. 你可以直接怎么用

1. 先只实现所有标记 `[MVP]` 的文件。
2. 每实现一个文件，对照本稿补“已实现/待实现”。
3. 任何新文件都按同样模板写：功能 + 调用 + 继承 + 信号。
